Speed Challenge

General information

In the speed challenge you will prove various skills in a range of small independent tasks.

You have exactly 1 hour to complete as many tasks as possible. The points you score in the speed challenge will get added to your total points of the whole competition.

Your solutions will be automatically marked on a central server in real-time. The first 3 competitors that complete all tasks get awarded some extra points (3 points for the winner, 2 points for the second place and 1 point for the third place). If time runs out before someone was able complete all tasks, the competitors with the most points are awarded the extra points. You can also continue working on the speed challenge if other competitors are already finished as those points will still get added to your total competition points.

All tasks can be tested by automated tests. To run them, change into their folder (e.g. cd /home/student/ICTT17/speedchallenge/work/task1) and execute the command stated in each task under expected result.
Before your start, please check that your name in /home/student/ICTT17/speedchallenge/competitor.json is correct.
If it is not, please call an expert.
You can only change existing files in the speedchallenge/work/task*/src/ folders (where * is a number). All other files will be reverted for the review process.

Task 4 — ASCII Art Checkerboard (JavaScript) medium

Write a JavaScript function that returns the textual representation of a square checkerboard.

The function should take into account two parameters, both are positive integers (≥ 0):

  • dim is the dimension of the checkerboard (the number of cases in one direction).
  • size is the dimension of a single case (how many characters wide and tall it should be).

The cases should be filled with a chequered pattern: half of the cases should be “empty” (they contain spaces), and the other half should be “filled” (they contain # characters). The top left case is always empty.

The borders around the cases should be drawn, using - characters for horizontal borders, | characters for vertical borders and + characters for intersections.

For example, with dim = 8 and size = 2, the checkerboard should look like this:

Task 7 — Newspaper Layout (HTML/CSS) hard

The task is to implement as specified a web page layout in CSS.

The HTML code of the page and its base styling (including colors, fonts, margins, borders, etc.) are already provided. You can add new CSS styles and edit the HTML structure of the page if you need (but the content should stay the same).

You have to implement the layout for the following parts:

  • Header info: The elements inside of the #header-info div should be vertically aligned. The space between the elements should be the same.
  • Content structure: The title, header info and footer should take the full width. Below the header, there should be two columns of the same height.
    The big article (#article-big) should be in the left column. It should be as wide as possible, and as tall as necessary for its contents.
    The advertisement (#advertisement-wrapper) and the small article (#article-small) should be in a 250 pixels-wide column on the right. The advertisement’s height should expand to take all the room that isn’t used by the small article.

Note: the requirements should still be met if the text changes (e.g. shorter or longer text content).

  • Working directory

    Implement the layout in: speedchallenge/work/task7/src/task7.css.

    The provided HTML code is available at speedchallenge/work/task7/src/index.html and you are allowed to edit it if you need to.

    You are not allowed to change any other file. All the other files will get reset for marking.

  • Expected result

    The layout was implemented as specified, and all integration tests run successfully
    (run npm test in the speedchallenge/work/task7 folder to open Cypress and run the tests)

Marking scheme

Objective 3

  • 1 The header info is implemented as specified, and the integration tests run successfully.
  • 2 The content structure is implemented as specified, and the integration tests run successfully.
